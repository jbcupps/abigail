# Vision Alignment Approval Checklist

Use this checklist before any chunk moves from planning to implementation.

## 1) Vision Completeness
- [ ] Chunk objective is explicit and bounded.
- [ ] Non-negotiables are listed.
- [ ] Preferred behavior is listed.
- [ ] Deferred questions are either resolved or intentionally carried forward.

## 2) Decision Quality
- [ ] Each major decision has a clear why.
- [ ] Trade-offs are documented.
- [ ] Lock type is defined (hard, soft, or mixed).
- [ ] Revisit triggers are defined.

## 3) Scope and Change Control
- [ ] File-level implementation scope is listed.
- [ ] Out-of-scope items are listed to prevent drift.
- [ ] Rollback path is documented.
- [ ] Dependency impacts (UI, commands, crates, docs) are identified.

## 4) Safety and Trust
- [ ] Trust/security implications are reviewed.
- [ ] Data handling implications are reviewed.
- [ ] Failure modes and fallback behavior are defined.
- [ ] Any destructive behavior requires explicit confirmation UX.

## 5) UX and Product Fit
- [ ] Copy/tone matches approved style for this chunk.
- [ ] User friction points are intentional and justified.
- [ ] Empty/loading/error states are defined.
- [ ] Accessibility implications are considered.

## 6) Verification Readiness
- [ ] Acceptance criteria are testable.
- [ ] Functional verification checklist is complete.
- [ ] Regression targets are listed.
- [ ] Performance targets (if relevant) are listed.

## 7) Approval Gate
- [ ] Decision brief reviewed with mentor.
- [ ] Mentor explicitly approves chunk for implementation.
- [ ] Any unresolved blockers are tracked with owner and next action.
